Vendor evaluation

Checklist for regulated operations browser automation vendors.

Use this checklist when evaluating automation for CRA background screening, FCRA adverse action, insurance eligibility, banking reconciliation, and other regulated portal workflows.

Back to overview

Core Evaluation Questions

  • Can the vendor operate the exact browser portals, legacy systems, files, and exception queues used by the team?
  • Can every task run return screenshots, downloaded artifacts, timestamps, portal messages, and structured outputs?
  • Can the automation separate deterministic portal work from policy decisions, approvals, and compliance judgment?
  • Can humans review exceptions before final action when data is missing, ambiguous, stale, or policy-sensitive?
  • Can the vendor integrate with the existing case queue, CRM, spreadsheet, n8n flow, Make scenario, Zapier Zap, or API router?

Evidence Requirements

Source Evidence

Portal screenshots, downloaded records, statements, notices, eligibility responses, and confirmation pages.

Run Evidence

Task id, run id, timestamps, input reference, credential context, status, retry count, and automation logs.

Review Evidence

Exception reason, reviewer handoff, final decision owner, policy note, and next action returned to the queue.
