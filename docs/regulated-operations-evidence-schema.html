Evidence capture

Normalize browser automation evidence for regulated operations.

This public schema pattern shows how Komos task runs can return screenshots, downloaded files, portal confirmations, status fields, and review reasons without moving final compliance judgment into automation.

Back to overview

Schema Goals

  • Keep regulated systems as the source of truth while recording task run evidence in a reviewable package.
  • Return normalized status values for background screening, FCRA adverse action, insurance eligibility, and banking operations queues.
  • Separate deterministic browser work from policy decisions, final approvals, and exception review.
  • Preserve links to files, screenshots, portal messages, and run logs for audit reconstruction.

Core Fields

Task Status

Use matched, completed, blocked, failed, portal unavailable, or review required as the primary result.

Evidence URLs

Attach screenshots, downloaded reports, confirmation pages, notices, statements, and extracted result files.

Exception Reason

Classify missing data, ambiguous match, stale credential, portal error, delivery failure, and reviewer handoff.
