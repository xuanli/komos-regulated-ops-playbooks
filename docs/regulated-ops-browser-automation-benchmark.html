What To Benchmark

Regulated operations teams should test more than whether an agent can click through a demo page. The useful benchmark is whether the system can complete real portal work, return auditable evidence, separate policy decisions from automation, and route exceptions to the right queue.

Benchmark Area What To Measure Public Komos Artifact
Portal execution Login handling, multi-step navigation, downloaded files, retries, and clear terminal states. Google Cloud Run task router
Evidence capture Screenshots, source documents, portal response text, timestamps, run ids, and reviewer notes. Evidence schema
Human review Exception reasons, missing data, ambiguous matches, policy-sensitive outcomes, and queue handoff. Partner review packet
Queue integration n8n, Make, Zapier, Airtable, API router, and case-management handoff patterns. Vendor evaluation checklist

Pass Criteria

  • The automation returns a structured status for every run: completed, needs review, blocked, or failed.
  • The output includes enough evidence for a reviewer to reconstruct the portal result without rerunning the task.
  • Policy decisions remain outside the automation unless an approved rule has already been encoded by the business.
  • Exceptions include a reason, the affected field, evidence links, and a recommended next action for the queue.
  • The system can be deployed behind the team's preferred queue, API router, or integration platform.
